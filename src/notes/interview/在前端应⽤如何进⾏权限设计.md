# 在前端应⽤如何进⾏权限设计

前端权限主要分为 `路由权限` 和 `功能点权限`，常见的模式是 `RBAC`，也有更灵活的 `ABAC`。**前端** 主要负责 **UI 控制**，**后端** 负责核心 **鉴权**。

路由权限：

- 静态路由 + 权限过滤（适合小型项目）

  * 前端定义完整的 **静态路由表**， 每个路由配置 `meta.permission`
  * 登录后从后端获取 **权限码/角色**，过滤出可访问的路由并渲染菜单。

- 动态路由 （适合大型项目）

  * 后端返回 **路由数据**（包含组件标识，权限信息等等）
  * 前端通过 **组件映射表** 结合路由数据组成对应的组件，通过 `router.addRoute` 动态注册。

功能点权限（菜单、按钮）：

实现方式：

- 权限指令：vue 可定义权限全局指令
- 渲染时判断：通过 **权限码数组** 判断是否渲染
- 交互限制：按钮显示但不可用

权限码：通常是类似 `user:add`、`order:delete` 的字符串标识

权限模式：

- `RBAC`（Role-Based Access Control）：用户 → 角色 → 权限（常见，维护成本低）
- `ABAC`（Attribute-Based Access Control）：基于用户属性、资源属性、环境条件进行判断（更灵活，如时间、地点、部门）
- `PBAC`（Policy-Based Access Control）：基于策略规则的权限系统（通常配合 JSON 配置或 DSL 规则引擎）

缓存与刷新策略

状态存储：

- 通过 **状态管理（vuex | pinia 等等）** 管理权限码
- 通过 `localStorage` 及获取、设置 api 进行管理权限码

刷新恢复：

- 直接使用 **状态管理** 或 **本地缓存的 get 方法** 获取
- 重新调用获取权限状态码的接口请求获取。

实现流程

（1）用户登录 → 后端返回 Token + 权限信息（权限码 / 角色）

（2）前端根据权限数据生成可访问路由表

（3）渲染菜单 & 按钮（通过权限指令 / 判断函数）

（4）调用 API 时，后端二次鉴权
